Documentatie Prototype Smart Hub

Opdracht: Maak een prototype van één gekozen library.

Tussen de code van het prototype staan overal notities die de code stap voor stap uitleggen.

Stap 1 - Extra oefenen

Github
Omdat ik nog niet eerder gewerkt heb met THREE.JS vóór het maken van deze specialisatie. Besloot ik om eerst nog een extra oefen opdracht te maken om de basics nog een keer te herhalen. Ik heb hier een basic scene opgezet waarin een witte vierkante ‘cube’ te zien is. Deze cube heeft een MeshLambertMaterial als material. Deze soort material kan ook licht weerkaatsen en maakt het dus mogelijk om lighting toe te voegen.

Stap 2 - Experimenteren

Github
Na het maken van een cube heb ik een aantal andere variaties uitgeprobeerd. Een voorbeeld hiervan is de TorusKnotGeometry.
Deze legt als het ware een knoop.

Stap 3 - Animeren

Github
Hierna ben ik begonnen aan het prototype. Ik heb online een model gevonden van een low poly paard samen met een model van een strand waar ik de animatie van het paard in kan zetten. Het paard heb ik zelf geanimeerd in blender.
Paard model
Strand model (uiteindelijk niet gebruikt)

Ik kwam er achter dat doordat ik te snel wilde gaan met het animeren en het instellen van de Scene, ik vastliep. Ik heb hulp gevraagd aan een van mijn groepsleden en ben overnieuw begonnen. Deze keer heb ik alleen de animatie van het paard gemaakt zonder de scene.
Nu dat het paard geanimeerd is kan ik beginnen aan het toevoegen van de animatie aan THREE.JS zodat hij weer te geven is.


Stap 4 - Implementeren

Github
Ik heb als eerste een standaard scene opgezet. Deze heb ik al tijdens mijn oefen opdracht gemaakt en kon ik nu mooi hergebruiken. Voordat ik de animatie in THREE.JS wilde zetten heb ik hem eerst in een gLTF Viewer gezet om te kijken of hij werkt naar behoren (gLTF is het bestandsformaat waarin de animatie word geexporteerd).

Deze werkte naar behoren en ik ben begonnen aan THREE.JS. Ik kreeg een aantal problemen met het weergeven van de animatie van het paard. Er alleen de skin van het model was te zien en de bone structure leek verdwenen. Het was alleen een losstaande afbeelding en geen animatie. Ik heb een beetje onderzocht hoe ik dit op kon lossen en heb deze discussion gebruikt als oplossing.

Na wat getweak aan mijn eigen code en het toevoegen van de correcte GLTFLoader(); ben ik er achter gekomen dat ik een ‘mixer’ miste, deze speelt de animatie af. Wat ik voorheen te zien kreeg was alleen een paard met zijn poten gebroken (zie afbeelding hieronder).
Verder heb ik ook nog controls toegevoegd met behulp van OrbitControls(); (die ik in week 2 tijdens het maken van de ShortList ook heb gebruikt).
Hierna heb ik het visueel aantrekkelijk gemaakt door kleuren toe te voegen aan de achtergrond.


Stap 5 - Verder uitwerken

Github
Om deze POC nog verder uit te werken richting ons project wilde ik eigenlijk nog meer dieren toevoegen. Uiteindelijk is dat binnen ons product ook mogelijk. Ik ben begonnen met een animatie van een kip die ik heb gevonden. Deze kip zelf animeren had me teveel tijd gekost.

Ik ben begonnen met een functie te schrijven die van model switcht wanneer er op de button geklikt word. Hij veranderd dan waar het model vandaan moet komen. Ben hierbij ook een probleem tegengekomen, hij veranderd niet. Ik denk zelf dat het te maken heeft met het updaten van de render.

Ondertussen ben ik een stap verder gekomen. Ook aangegeven in Git. Op dit moment start hij een nieuwe Scene onder de al bestaande Scene op de pagina waardoor de Scenes dus gestacked worden onder elkaar binnen de pagina.

Na veel rond gesnuffel en geprobeer heb ik voortgang geboekt. Het is me gelukt om meerdere animations in 1 scene te zetten (zie afbeelding rechts). Nu proberen om de vorige animatie uit de scene te zetten voordat de nieuwe wordt toegevoegd.

Ook is inmiddels de laatste stap gezet. Het probleem van het laden van meerdere objecten binnen één scene is verholpen. De Init(); ‘refresht’ nu als het waren elke keer wanneer er op een knop word gedrukt. Het pad van de gekozen animatie word meegestuurd als parameter.
